# Agent evaluations for Microsoft 365 Copilot agents

> [!IMPORTANT]
> This tool is currently available in private preview. The instructions in this article apply to the private preview experience.

Agent evaluations enable you to test and assess the performance of Microsoft 365 Copilot agents by sending prompts to a deployed agent and automatically scoring the responses using Azure AI Evaluation metrics. 

The Microsoft 365 Copilot Agent Evaluations CLI provides a zero-configuration command-line interface that allows you to:

- Send prompts to a configured agent chat API endpoint  
- Collect agent responses  
- Automatically evaluate responses using Azure AI Evaluation SDK metrics such as:
  - Relevance  
  - Coherence  
  - Groundedness  
  - Tool call accuracy  
  - Citations   

You can run evaluations by submitting a batch of prompts or interactively entering prompts to test agent responses in real time. 

## Evaluation workflow

Agent evaluations follow a structured workflow:

1. Submit prompts to a configured agent chat API endpoint  
2. Collect responses generated by the agent  
3. Evaluate responses locally using the Azure AI Evaluation SDK  
4. Generate an evaluation report containing per‑prompt scores and aggregate metrics   

This process enables you to measure the quality and reliability of agent responses across defined test scenarios.

## Evaluation inputs

You can provide prompts to the evaluation CLI using multiple input methods, including:

- A list of prompts provided at the command line  
- A JSON file containing prompt and expected response pairs  
- Interactive prompt entry during runtime   

When you run the evaluation tool, it automatically searches for prompt files in the following locations:

- The current project directory  
- The `./evals/` subdirectory   

If no prompts file is found, the CLI can generate a starter file with example prompts that you can edit for your evaluation scenarios.   

## Evaluation outputs

Evaluation results are saved automatically to an HTML report in the following format:

`./evals/YYYY-MM-DD_HH-MM-SS.html`

The report includes:

- Relevance score  
- Coherence score  
- Groundedness score  
- Per‑prompt evaluation details  
- Aggregate evaluation metrics   

You can also export evaluation results to:

- JSON  
- CSV   

These outputs enable you to analyze evaluation results programmatically or integrate them into reporting workflows.

## Evaluation datasets

Evaluation prompt files must include the following fields:

- `prompt`  
- `expected_response`   

Evaluation documents support schema versioning and should include a `schemaVersion` field and an `items` collection that defines the evaluation prompts and expected responses.   

Within a major schema version, backward compatibility is guaranteed for evaluation documents.   

## Next steps

To begin evaluating your agent, install and configure the Microsoft 365 Copilot Agent Evaluations CLI in your agent project directory.
